üîç Key Concepts Explained

Immutable by Default

Variables are immutable unless explicitly marked with mut. This prevents accidental modifications and makes code safer.

Type Inference

Ovie automatically determines variable types based on assigned values, reducing verbosity while maintaining type safety.

Natural Syntax

Variable declarations use simple, readable syntax without complex type annotations unless needed.

Scope Rules

Variables have clear scope boundaries - local variables are contained within functions, global variables are accessible everywhere.

üìö Variable Types in Ovie

1. Numbers

Ovie uses a unified Number type that handles both integers and floating-point values automatically:

mut age = 25        // Integer
mut height = 5.9    // Float
mut result = age + height  // Mixed operations work seamlessly

2. Strings

Text data with support for concatenation and interpolation:

mut name = "Alice"
mut greeting = "Hello, " + name + "!"  // Concatenation

3. Booleans

True/false values with logical operations:

mut is_active = true
mut can_proceed = is_active && age >= 18

4. Arrays (Coming in next examples)

Collections of values that will be covered in detail in the arrays example.

üõ°Ô∏è Memory Safety

Ovie's variable system provides memory safety through:

  • Automatic Memory Management: No manual memory allocation/deallocation
  • Immutability by Default: Prevents accidental data corruption
  • Scope-based Cleanup: Variables are automatically cleaned up when out of scope
  • Type Safety: Prevents type-related errors at compile time
